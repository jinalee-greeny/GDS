# Foundations Token Studio — Figma Plugin

Generated by `python3 build_apps.py` (edit `core/*` and `plugin/*.template.*` / `plugin/code.src.js`, never the generated `code.js`/`ui.html`).

## Load in Figma (development)
1. `python3 build_apps.py` from the repo root.
2. Figma desktop → Menu → Plugins → Development → Import plugin from manifest…
3. Select `plugin/manifest.json`.
4. Run: Plugins → Development → Foundations Token Studio.

## Use
- Tweak tokens in the panel (same editor as the web app).
- Under "Figma Variables", enable the target and check the groups to export.
- Click "Figma에 적용" (always visible at the bottom) → writes every enabled target.
- Re-applying updates in place (no duplicates).

## Styles (Text + Effect)
- Under "Text Styles", check the weights to generate and set the font family (must be a font available in your Figma — default 'Pretendard'). Under "Effect Styles", toggle shadow export.
- "Figma에 적용" (always visible at the bottom) writes the enabled targets.

## Manual QA checklist
- [ ] Panel renders (controls + preview + WCAG) inside Figma.
- [ ] Apply with all groups → `Foundations` collection created with color/*, space/*, radius/*, etc. of correct types.
- [ ] Colors resolve to the expected swatches; spacing/radius are FLOATs; fontFamily/easing are STRINGs.
- [ ] Re-apply after changing a hue → variables update, count = updated (no new dupes).
- [ ] Deselect some groups → only selected groups written.
- [ ] Result summary shows 생성/갱신/실패 counts.

### Manual QA (Figma desktop)
- [ ] The "Figma에 적용" button is always visible (fixed bottom bar), no scrolling needed.
- [ ] Apply with Variables + Effect Styles → Foundations variables + shadow/* effect styles created.
- [ ] Apply with Text Styles (regular+bold, family that exists) → text/<size>/<weight> text styles created.
- [ ] Set the font family to one NOT installed → those text styles report a clear "폰트 사용 불가" failure; others still succeed.
- [ ] Re-apply → styles update in place, no duplicates. Result summary shows 생성/갱신/실패.

## Note
- Changing a token's underlying variable TYPE (COLOR/FLOAT/STRING) causes the variable to be removed and recreated with a new ID, which can orphan any existing Figma layer/variable bindings to that token. This is expected Figma-API behavior, noted so users aren't surprised.
